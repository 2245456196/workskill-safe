# Contributing

## Scope

Contributions should improve privacy detection, deterministic sanitization, audit reporting, Agent Skill scaffolding, validation, tests, documentation, or developer workflow.

## Data rule

Use synthetic data only. Never paste real work notes, employer data, client data, personal data, authentication material, private links, or internal paths into Issues, Discussions, commits, tests, or pull requests.

## Before opening a pull request

- Search existing Issues.
- Link behavior changes to an Issue.
- Add tests for new rules.
- Include a positive case, a near miss, and false-positive considerations.
- Explain privacy impact.
- Update documentation when behavior changes.

## Pull request checklist

- [ ] Synthetic fixtures only
- [ ] No company, client, personal, authentication, or private project data
- [ ] Tests added or updated
- [ ] Privacy impact described
- [ ] Documentation updated
- [ ] Changelog updated for user-visible behavior
