# 組合識別人工檢查表

自動掃描只能處理明確格式，無法判斷多個普通資訊組合後，是否會讓外部人員推回原始公司、專案、人物或事件。

公開 Skill、範例、Pilot 結果或維護紀錄前，需完成人工檢查。

## 組合資訊

- [ ] 產業與少見服務細節已泛化。
- [ ] 精確日期、季度與事件順序已泛化。
- [ ] 精確金額、數量、比例與價格區間已移除或替換。
- [ ] 地點與少見營運範圍已泛化。
- [ ] 職稱與關係描述無法指向特定人物。
- [ ] 產品名稱、內部縮寫與少見流程用語已移除。
- [ ] 利害關係人、技術與流程的組合無法辨識來源。

## 來源重建

- [ ] 讀者無法重建原始會議、提案或專案時序。
- [ ] 特殊語句無法透過搜尋連回來源文件。
- [ ] 範例沒有保留可辨識的決策或質疑順序。
- [ ] 內容保留的是可重複方法，而非變相的個案摘要。

## 發布判定

只要審查者仍能合理推測原始情境，就停止發布並重新泛化。

自動掃描結果與人工複核結果必須分開記錄。
