# 隱私與機密操作指南

## 核心規則

公開 GitHub 只放程式、規則、公開文件、完全虛構測試資料，以及無法反向辨識的聚合結果。

原始工作資料必須留在私人、本機環境。

## 絕對不可公開

- 公司計畫書、會議紀錄、報價、成本、分潤與合約
- 客戶名單、合作名單、洽談狀態與未公開產品規格
- 姓名、電話、Email、地址、生日、身分或帳戶資料
- 內部網址、文件連結、通訊群組、主機與私人路徑
- 認證資訊與本機設定機密

## 去識別化不只是刪除姓名

仍需處理公司別稱、產品名稱、精確日期、金額、地點、角色、內部縮寫、文件路徑與少見流程。

## 發布前人工檢查

- [ ] 能否從內容猜出公司或客戶？
- [ ] 精確日期、地區與金額組合後是否可識別？
- [ ] 是否仍有私人網址、路徑或帳號資訊？
- [ ] 是否包含未公開的商業判斷？
- [ ] 是否只留下可重複的方法，而非真實事件？

## Fail-closed 原則

規則檔缺失、尚未人工確認、仍含私人路徑或未處理敏感資訊時，工具應拒絕生成 Skill。
