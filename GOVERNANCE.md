# Governance

## Maintainer model

The project begins with one primary maintainer responsible for roadmap, Issue triage, pull request review, privacy policy, releases, security response, and contributor guidance.

## Decision principles

1. Privacy over convenience
2. Deterministic behavior over opaque automation
3. Small scope over feature accumulation
4. Evidence over claims
5. Public reproducibility over private anecdotes

## Changes requiring explicit maintainer approval

- network access
- external service integration
- telemetry
- cloud processing
- data retention
- changes to privacy defaults
- automatic publication
- release publication
