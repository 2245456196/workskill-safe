# Proposal Output Schema

Use these sections in this order.

## 1. Executive decision summary

Required fields:

- problem
- recommended choice
- rationale
- main risks
- decision required

## 2. Stakeholder matrix

| Stakeholder | Problem | Adoption incentive | Main objection | Required response |
|---|---|---|---|---|

## 3. Service and technical scope

Required fields:

- user workflow
- service modules
- technical modules
- ownership boundaries
- data boundaries
- excluded scope

## 4. MVP choice

Required fields:

- selected option
- alternatives considered
- why this option is smaller and testable
- input
- output
- success criteria
- failure criteria

## 5. Business assumptions

| Assumption | Status | Validation method | Decision impact |
|---|---|---|---|

Allowed status values:

- verified
- assumption
- pending validation
- rejected

## 6. Claim-to-evidence table

| Claim | Evidence | Source status | Confidence | Missing data |
|---|---|---|---|---|

Never create precise values without a source.

## 7. Reviewer objections

| Question | Response direction | Evidence required | Structural revision |
|---|---|---|---|

## 8. Slide outline

For each page provide:

- page title
- one core judgment
- supporting evidence
- required visual

## 9. Missing information and risks

Separate:

- information gaps
- technical risks
- operating risks
- adoption risks
- publication risks

## 10. Action list

| Action | Owner role | Dependency | Due status | Completion evidence |
|---|---|---|---|---|

## 11. Publication checklist

- automated scan completed
- human contextual review completed
- unsupported claims labeled
- identifying combinations generalized
- source authorization confirmed
